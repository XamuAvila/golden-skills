# Variable Reward — Phase 3 of the Hook Model

> Source: Eyal, *Hooked*, Ch. 4. Sub-skill of `hooked-habit-design`.

What separates a Hook from a plain feedback loop is the ability to **create craving**. Predictable
loops (the fridge light) create no desire; **add variability** and intrigue is created. Dopamine
rises when the brain *anticipates* the reward; variability multiplies the effect.

Master rule: the reward must **satisfy the need AND leave the user wanting to re-engage**.

## The 3 types of variable reward

| Type | What it seeks | Book examples | Product example |
|------|---------------|---------------|-----------------|
| **Tribe** | **Social** rewards — connection, acceptance, reciprocity | Likes, Quora upvotes, Facebook | Kudos/recognition among teammates; comments and reactions |
| **Hunt** | **Material/information** resources — the "chase" | Twitter feed, Pinterest scroll, slot machine | A feed of new leads/items; searching results; a novelty digest |
| **Self** | **Intrinsic** rewards — mastery, competence, completion | Leveling up (games), "inbox zero" (Mailbox), Codecademy | Clearing a task queue; completing a checklist; a personal goal bar |

**A strong product usually combines all three** (e-mail uses Tribe + Hunt + Self). When designing,
ask which type(s) the feature activates.

## Finite vs infinite variability

- **Finite variability**: becomes predictable with use → loses appeal (Breaking Bad once watched;
  re-skinned FarmVille → Zynga dropped 80%). Viable business, but requires *churning out new content
  nonstop* ("studio" model).
- **Infinite variability**: sustains variability with use → holds the user (multiplayer World of
  Warcraft; user-generated content: YouTube, Pinterest, Dribbble). **The unpredictable behavior of
  other people is the best source of infinite variability.**

→ Rewards that depend on **people** (new messages, leads, reviews, teammate activity) have more
durable variability than fixed system rewards.

## Autonomy — don't threaten it, or *reactance* follows

When autonomy is threatened, the user feels forced and **rebels** (reactance). MyFitnessPal became
"MyFitnessPain" (obligation → abandonment); Quora forced opt-in on "views" and there was a revolt.
Fitocracy won by leveraging behavior the user **already wanted** to do.

- The "**but you are free**" technique: reaffirming freedom to choose **doubles** compliance
  (meta-analysis, 42 studies). → Prefer "whenever you want, you can…" to flows that *force*.
- Present an **implicit choice** between the old way and a new, more convenient one — don't force the
  new behavior.

## Gamification is not fairy dust

Points/badges/leaderboards **only work if they scratch the user's real itch**. Mahalo paid in cash
and lost users; Quora gave social recognition (upvotes) and retained. If there is a mismatch between
the user's pain and the assumed solution, **no gamification will save it**.

## Phase checklist
- [ ] Which of the 3 types — Tribe / Hunt / Self — does the reward activate?
- [ ] Does the reward **satisfy and leave them wanting more** (has mystery/variability)?
- [ ] Is the variability **infinite** (people/content) or finite (will saturate)?
- [ ] Did I preserve the user's **autonomy** (no forcing; reaffirm freedom of choice)?
- [ ] If I used gamification, does it scratch the real itch — not just decoration?
